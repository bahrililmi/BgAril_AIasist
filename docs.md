## 🗃 2. Database Schema (Supabase PostgreSQL)

### Table: `messages`
- `id`: uuid (Primary Key, default: uuid_generate_v4())
- `created_at`: timestamptz (default: now())
- `group_name`: text (Nama grup Telegram)
- `category`: text (Kategori hasil mapping, misal: 'ZONA_WARRIOR')
- `sender_name`: text (Nama pengirim)
- `sender_username`: text (Username pengirim, nullable)
- `message_body`: text (Isi pesan teks)
- `has_media`: boolean (Apakah pesan mengandung foto/dokumen)
- `is_ai_processed`: boolean (default: false - untuk trigger automasi n8n nantinya)

### Table: `groups_config`
- `id`: uuid (Primary Key)
- `group_name`: text (Unique - Nama grup persis di Telegram)
- `category`: text (Dropdown: ZONA_WARRIOR, NEED_KANVAS, H2H_SUPPLIER, ROOFTOP, RMW_INTERNAL, UNMAPPED)
- `is_active`: boolean (default: true)

## 📡 3. Backend Logic (Python Collector - `collector.py`)
Script ini berjalan sebagai daemon 24/7 di VPS aaPanel.

**Execution Flow:**
1. Load `.env` (TELEGRAM_API_ID, TELEGRAM_API_HASH, SUPABASE_URL, SUPABASE_KEY).
2. Initialize `supabase` client.
3. Initialize `TelegramClient` (Telethon) menggunakan local session file.
4. Listen to `events.NewMessage`.
5. **Interceptor Logic:**
   - Ambil `event.chat.title` (group_name). Jika private chat, skip.
   - Ambil `event.text` (jika kosong dan tidak ada media, skip).
   - Fetch list grup dari Supabase tabel `groups_config`.
   - Lakukan pencocokan `group_name`. 
   - Jika grup tidak ada di DB, insert ke `groups_config` dengan status `category = 'UNMAPPED'` dan `is_active = true`.
   - Jika grup ada dan `is_active == true`, insert detail pesan ke tabel `messages`.
   - Gunakan `try-except` block yang solid agar script tidak mati jika terjadi gagal koneksi database sementara.

## 💻 4. Frontend Application Logic (Next.js 14)

### Rule of Thumb untuk AI:
- Gunakan **Server Actions** (`/actions/*.ts`) untuk semua operasi database (fetch, insert, update). Dilarang menggunakan API Routes untuk komunikasi internal database.
- Gunakan **Client Components** (`"use client"`) HANYA pada komponen yang butuh interaktivitas (seperti shadcn Tabs, form submit, onClick event, dan DataTable pagination).
- Pisahkan UI table (`DataTable`) ke dalam komponen mandiri di folder `/components`.

### Page Specifics:
1. **/settings (Client Component with Server Actions):**
   - Tab "Groups": Panggil `getGroups()` dari Server Actions. Tampilkan di DataTable. Kolom 'Category' harus berupa shadcn `Select` dropdown agar admin bisa langsung mengubah kategori, memanggil fungsi `updateGroupCategory(id, newCategory)` saat value berubah.
   
2. **/messages (Server Component to Client Table):**
   - Fetch data 500 pesan terakhir secara server-side.
   - Lempar data ke Client Component `<MessageTable data={data} />` (menggunakan shadcn DataTable) agar bisa di-sort dan filter berdasarkan kolom `category`.

3. **/ai-query (Client Component):**
   - UI Chat sederhana (mirip ChatGPT).
   - Saat user submit (misal: "Buatkan rangkuman lapangan hari ini"), UI memanggil **Route Handler** (`/app/api/gemini/route.ts`).
   - Route handler ini akan: 
     a. Query database Supabase untuk mengambil pesan hari ini dari `messages`.
     b. Merakit *prompt* sistem + data JSON + pertanyaan user.
     c. Hit Google Gemini API (menggunakan package `@google/generative-ai`).
     d. Kembalikan response text ke UI.

## 🚀 6. Deployment Architecture (Wajib Diperhatikan AI)
Project ini mengadopsi arsitektur terpisah (decoupled) untuk efisiensi resource. AI wajib memastikan kode yang dibuat kompatibel dengan environment berikut:

### A. Frontend (Web/Next.js) -> Hosted on VERCEL
- **Platform:** Vercel (Hobby/Free Tier).
- **Karakteristik:** 
  - Tidak menggunakan custom server (`server.js`). 
  - API Routes (`/app/api/gemini/route.ts`) akan berjalan sebagai Vercel Serverless Functions.
  - Environment variables (`SUPABASE_URL`, `SUPABASE_ANON_KEY`, `GEMINI_API_KEY`) akan di-set langsung melalui Vercel Dashboard.

### B. Backend Collector (Python) -> Hosted on VPS aaPanel (Ubuntu)
- **Platform:** VPS Ubuntu dengan aaPanel control panel.
- **Karakteristik:** 
  - Script `collector.py` berjalan sebagai *background daemon* (24/7) menggunakan **Python Manager** di aaPanel atau **PM2**.
  - Script tidak membuka port HTTP (bukan Flask/FastAPI), melainkan murni berjalan sebagai *listener* Telegram (Telethon) dan langsung menembak data ke Supabase Cloud.
  - Wajib memiliki implementasi *auto-reconnect* jika koneksi Telegram atau Supabase sempat terputus.

### C. Database -> Hosted on Supabase Cloud
- Menggunakan layanan Supabase Cloud (PostgreSQL as a Service). Tidak di-host lokal di aaPanel untuk mengurangi beban VPS.
